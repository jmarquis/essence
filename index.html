Minimalist

No dependencies. No frameworks. No clunky CSS classes. In fact, no CSS output at all until you actually use the mixins. It’s just a single .less file with a simple set of mixins.

Responsive

Completely fluid, completely flexible. Use fractions and percentages to define consistent layouts. Set custom gutters or breakpoints. Use media queries to adjust for different sizes. It’s responsive layouts done right.

Semantic

No layout-related CSS classes. Keep your styles in your stylesheets. Apply layouts styles using single-line mixins and keep your markup squeaky clean.

Usage

Basic Layout

@import "essence";

// choose a container...
.content {

	// ...start a layout...
	.essence();

	article {
		// ...and start defining cells
		.cell( (2/3) );
	}

	aside {
		.cell( (1/3) );
	}
}

Custom Responsive Behavior

By default, cells stack below 768px. You can specify an alternate stack point or disable stacking altogether as shown below. 

@import "essence";

.content {

	.essence();

	section {

		// create six equal-sized columns
		.cell( (1/6); @breakpoint: 0px );

		// custom rearrangement below 1024px
		@media (max-width: 1023px) {
			.cell( (1/3); @breakpoint: 0px );
		}

		// custom rearrangement below 768px
		@media (max-width: 767px) {
			.cell( (1/2); @breakpoint: 0px );
		}
	}
}
