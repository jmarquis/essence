essence

minimalist, responsive, semantic grids

v0.1.0 GitHub

Grids done right

There are plenty of grid systems on the web. Essence gets you what you need without the clutter by taking a minimalist approach. It won’t do everything — instead, it provides a solution that’s perfect for most cases and extensible for others. Use it as it ships for simple, flexible grid layouts, or use it as a boilerplate for a more complex custom solution. Think of it as a starting point for building amazing grids with LESS. 

@import "essence";

.content {
    #grid > .row(12);

    article {
        .cell(8);
    }

    aside {
        .cell(3);
        .offset(1);
    }
}

Minimalist

No dependencies. No frameworks. No clunky CSS classes. In fact, no CSS output at all until you actually use the grid mixins. It’s just a single .less file with a single #grid namespace.

Responsive

Completely fluid, completely flexible. Use any container of any size as a grid row. Define different numbers of columns for each row, or custom gutters or breakpoints. It’s responsive grids done right.

Semantic

No layout-related CSS classes. Keep your styles in your stylesheets. Apply grid layouts using single-line LESS mixins and keep your markup squeaky clean.

Usage

Grab the latest version, include the file in your LESS stylesheet, and start making grids! 

@import "essence";

// use any container as a row
.some-container {
    // define any number of columns for a row
    #grid > .row(16);

    // or use the default defined in essence.less
    #grid > .row();

    // or specify custom values for column count, gutter size and breakpoint
    #grid > .row(10, 20px, 800px);

    // use any direct children as cells
    .some-child {
        .cell(6);
    }

    // cells can have offsets
    & > .another-child {
        .cell(4);
        .offset(2);
    }

    // layouts can change at different sizes
    & > .sidebar {
        .cell(3);
        .offset(1);

        @media (max-width: 1024px) {
            .cell(4);
            .offset(0);
        }
    }
}

Settings

The top portion of essence.less contains default values for a row’s column count, gutter size and break point. These are used anytime a #grid > .row() mixin is used without specifying any of the respective parameters.

Advanced Customization

Essence was designed to be minimalist, and as such the codebase is relatively small and simple. Adding custom functionality or changing existing behavior is possible, but may result in problems when upgrading to newer versions of Essence. Customize at your own risk!

So... Really? Another grid system?

Essence was born out of a need to quickly develop sites from grid-based Photoshop designs. Because good designers take liberties in crafting awesome designs, there are always cases where simple grid solutions just aren’t flexible enough. More complex frameworks are typically overkill — while they can be good for some cases, they tend to include tons of features and CSS clutter that goes unused.

Essence strikes a balance between simplicity and flexibility with the power of LESS. Use the features you need, ignore the ones you don’t. No code is wasted.
